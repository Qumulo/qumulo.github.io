Qumulo Resources on GitHub

Qumulo creates products that to let users manage their unstructured file data for high-performance workloads at exabyte scale. This page is your starting point for Qumulo resources on GitHub.

Documentation on GitHub

To provide the most up-to-date documentation on Qumulo products, we have started a Docs as Code repository on GitHub, where you can use, ask questions, and make suggestions for our documentation.

Getting Started

Qumulo's scale-out data storage solution is enhanced with our RESTful API. The API is built right into our file system. It's the foundation used behind the Qumulo web application and is used internally by our engineering team for testing, automation, and more. As a storage admin or storage user, you can use this programmatic power to:

  • Automate tasks like creating shares, quotas, or snapshots
  • Streamline your workflow with scripted automation
  • Dive deeper into analytics to understand how your storage is being used

To get started, see Introduction to the Qumulo API, where you can learn about the API's architecture and try out the following implementations.

  • python bindings: This is the main method of interacting with the Qumulo API programmatically.
  • qq CLI tool: This wrapper that provides a streamlined command-line interface for the API.
  • curl commands: These examples explain the raw REST elements that back the API.

Qumulo Github Repositories

Our GitHub repositories are organized into categories with code examples that demonstrate a variety of uses of the Qumulo API.

API-Driven Applications

  • SNMP Agent: A Simple Network Management Protocol (SNMP) agent that reports on node and drive states for a Qumulo cluster.
  • File Upload Web App: A Flask-based Python application that enables user home directories, file upload, and management functionality.
  • Daily Storage Reports Web App: A Flask-based application that creates reports for store, present, and alert on capacity and IOPS usage by path, over time.
  • Capacity Email Alerts: A Python application that sends emails when paths exceed capacity thresholds.
  • API metrics to CSV, ElasticSearch, Influx, and Splunk: A script that monitors workload data, including paths, clients, IOPS, and throughput.

Power Tools

  • Disk Usage (du): Use Qumulo's aggregates to see the usage of a full file tree in real time.
  • Parallel Copy (rsync: Use capacity aggregates to divide a large data copy job to run with rsync.
  • Capacity by User: Use the file samples API to provide quick estimates of storage usage by user, and to estimate costs.
  • Access Control List (ACL) Management: Manage your ACLs.
